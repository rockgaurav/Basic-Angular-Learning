ngOnDestroy
What is an ngOnDestroy?

ngOnDestroy calls before component destroy.

Use: 1. Use to Unsubscribe from Observables, detach event handlers, or clean up resources to prevent memory leaks.

2. API subscription unsubscribe

3. Remove event listener or memory.

4. If you are not using any API or event listener, then no need to use.

Real Scenario: A timmer is running in component and must to close while page change.

Not Use ngOnDestroy:
  1. 1. Memory leak is possible and app also get slow.
  2. 2. Sometime unwanted behaviour also show like (old data show, console errors).
  3. 3. Intervals, subscriptions ya event listeners continue run either component destoryed.
