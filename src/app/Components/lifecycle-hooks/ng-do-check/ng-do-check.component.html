ngDoCheck
What is an ngDoCheck?

ngDoCheck is triggered after the content (ng-content) has been projected into view. It is called only once after the first ngDoCheck.

Use: 1. When using content projection (ng-content).

2. Data bind with Either Event or User action.

Real Scenario: It detect complex object changes which not detected by ngOnChanges() means deep check.

Not Use ngDoCheck:
  1. Angular will update but not showing change value.
  2. Not able to run custom logic.
